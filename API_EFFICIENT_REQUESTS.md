# Более эффективные API для отчётов по счёту

Для запросов вида «финансовый результат за период» или «сводка по операциям» вместо постраничного обхода **GetOperationsByCursor** есть варианты эффективнее.

## 1. Брокерский отчёт (GetBrokerReport)

**Назначение:** готовый отчёт по сделкам за период (покупки/продажи, комиссии брокера и биржи, НКД и т.д.). В документации T-Invest рекомендуется для точных данных по инструментам после корпоративных действий.

**Схема работы (два шага):**

1. **GenerateBrokerReport** — запуск формирования отчёта.
   - Параметры: `account_id`, `from`, `to` (период по UTC).
   - Ответ: `task_id` (идентификатор задачи).

2. **GetBrokerReport** — получение готового отчёта по `task_id`.
   - Параметры: `task_id`, `page` (нумерация с 1).
   - Ответ: массив записей **BrokerReport** (сделки: trade_id, figi, direction, total_order_amount, broker_commission, exchange_commission и др.), плюс `itemsCount`, `pagesCount`, `page`.

**Плюсы:** один раз сформировали отчёт за период — дальше только забирать по страницам; данные по сделкам уже в формате отчёта.  
**Ограничение:** в отчёт входят **сделки** (торговые операции). Дивиденды, купоны, пополнения/выводы — это не сделки; для полного «доход/расход» их по-прежнему берут из операций (см. ниже).

**Документация:** [GetBrokerReport](https://developer.tinkoff.ru/invest/api/operations-service-get-broker-report), [investAPI Operations](https://tinkoff.github.io/investAPI/operations/) (сообщения GenerateBrokerReportRequest, GetBrokerReportRequest, BrokerReport).

---

## 2. GetOperationsByCursor с limit=1000

Текущий способ (операции по счёту) остаётся единственным для **всех** типов операций: пополнения, выводы, дивиденды, купоны, налоги, комиссии, покупки, продажи.

**Оптимизация:** указывать **limit=1000** (максимум в API) — меньше запросов при обходе по `next_cursor`. В TInvestMcp в вызов можно передать `limit`: 1000.

---

## 3. GetOperations (без курсора)

Метод **GetOperations** (без ByCursor) возвращает список операций за период одним ответом (без пагинации по курсору).

**Особенности (из документации):**
- По опционам данные могут отображаться некорректно; для опционов рекомендуют GetOperationsByCursor.
- Ограничения на объём ответа в документации явно не указаны; при большом числе операций возможны таймауты или ограничения размера ответа.

Для «умеренного» числа операций за год один запрос GetOperations может быть удобнее, чем несколько GetOperationsByCursor. При очень большом количестве операций надёжнее курсор с limit=1000.

---

## Итог

| Задача | Рекомендация |
|--------|--------------|
| Точные данные по **сделкам** (покупки/продажи) за период | GenerateBrokerReport → (подождать) → GetBrokerReport по task_id |
| **Все** операции (в т.ч. дивиденды, купоны, ввод/вывод) за период | GetOperationsByCursor с **limit=1000** и обход по next_cursor |
| Небольшой объём операций за период | Попробовать GetOperations (один запрос) |

Добавление инструментов **tinvest_generate_broker_report** и **tinvest_get_broker_report** в TInvestMcp позволит запрашивать брокерский отчёт из Cursor без ручных вызовов API.
